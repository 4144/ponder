What?

C++ type reflection library for exposing type data in C++11 programs.

About

Purpose of this library

What is reflection?

Reflection allows a program to inspect its own type data at runtime. For example, this is useful for data-driven programs: we might create classes from descriptions in data read at runtime.

Fork of CAMP

Ponder continues the from CAMP, removing the Boost dependency, instead, using C++11 features. The aim here is to allow simplification using C++11, and improve build times.

Simple Example

{% highlight cpp %} class Adder { public: Adder() : total_(0) {} void add(int value) { total_ += value; } int getTotal() const { return total_; } private: int total_; }; {% endhighlight %}
A C++ class that we would like to expose for reflection.
{% highlight cpp %} ponder::Class::declare("Adder") .constructor<>() .function("add", &Adder::add) .property("total", &Adder::getTotal); {% endhighlight %}
How we would like to expose the class using Ponder. We expose a constructor, an "add" function, and a "total" property.
{% highlight cpp %} auto& metaclass = ponder::classByName("Adder"); // find the class auto adder = metaclass.construct(); // create an instance adder.call("add", 7); // call a function adder.call("add", 5); int total = adder.get("total"); // get a property -> 12 {% endhighlight %}
Now we can make runtime instances of the object and manipulate them. See documentation for more details.
